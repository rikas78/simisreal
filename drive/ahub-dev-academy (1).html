Missione 1 — Cambia il Titolo dell'App
Impari: HTML · Struttura di una pagina web · Tag
HTML ›
🎯 Obiettivo App
Vuoi cambiare il titolo della scheda del browser da "A-HUB | Alessio Business" a qualcosa di tuo. Tipo "RESELL KING 👑".
📚 Concetto: HTML
1
HTML è come lo scheletro di una pagina web. Ogni pezzo è avvolto in un tag, tipo: <h1>Titolo</h1>
2
I tag si aprono <tag> e si chiudono </tag>. Tutto il contenuto va in mezzo.
3
Il tag <title> controlla il nome nella scheda del browser.
Dove cercare nel file
HTMLahub-v5.html — riga 4
<!-- Trova questa riga -->
<title>A-HUB | Alessio Business</title>

<!-- Cambiala in qualcosa di tuo, es: -->
<title>RESELL KING 👑 | By Alessio</title>
✅ Risultato
Apri il file nel browser — la scheda in alto mostrerà il tuo nuovo titolo.
⚡ Sfida Extra
Trova il tag <h1> e <h2> nel file. Cosa contengono?
Spoiler: i numeri dopo h (h1, h2, h3...) indicano l'importanza del titolo. h1 = più importante.

Missione 2 — Cambia il Colore dell'App
Impari: CSS · Colori · Classi · Proprietà
CSS ›
🎯 Obiettivo App
Il viola/indigo di A-HUB è bello, ma tu vuoi il rosso. O il verde. O qualsiasi colore tu voglia — è il tuo brand!
📚 Concetto: CSS
1
CSS è lo stile della pagina. Se HTML è lo scheletro, CSS è i vestiti.
2
Si scrive così: proprietà: valore; — tipo color: red; o background: #FF0000;
3
I colori si scrivono in HEX: ogni colore è un codice con #. Es: #EF4444 = rosso, #10B981 = verde, #F59E0B = giallo.
Come fare
CSSnel tag <style> in cima al file
/* Cerca tutte le occorrenze di #4F46E5 */
/* e sostituiscile con il tuo colore */

/* Esempi di colori belli: */
--rosso:    #EF4444;
--verde:    #10B981;
--arancio:  #F97316;
--rosa:     #EC4899;
--oro:      #F59E0B;
Sito utile
Vai su coolors.co — genera palette di colori fantastici. Premi SPAZIO e scegli quello che ti piace 🎨
⚡ Sfida Extra
Cambia anche il colore di sfondo dell'app. Cerca background:#F1F5F9 e mettilo più scuro, tipo #111827.
Cosa succede? L'app diventa dark mode 🖤

Missione 3 — Aggiungi una Categoria
Impari: JavaScript · Variabili · Array
JS ›
🎯 Obiettivo App
Le categorie di A-HUB sono: Magliette, Felpe, Cappellini, Sneakers, Accessori. Vuoi aggiungere "Orologi ⌚" o "Borse 👜".
📚 Concetto: Variabili e Array
1
Una variabile è una scatola con un nome che contiene un valore. let nome = "Alessio";
2
Un Array è una lista di cose: let lista = ["cosa1", "cosa2", "cosa3"];
3
Per aggiungere un elemento si usa .push("nuovaCosa") oppure si scrive direttamente nella lista.
Dove cercare
JavaScriptnel tag <script> in fondo al file
// Trova questa riga — è un Array di categorie
const CATS = [
  'Magliette',
  'Felpe',
  'Cappellini',
  'Sneakers',
  'Accessori'
];

// Aggiungine una nuova:
const CATS = [
  'Magliette',
  'Felpe',
  'Cappellini',
  'Sneakers',
  'Accessori',
  'Orologi ⌚'  // ← aggiunta!
];
✅ Risultato
Il filtro "Orologi" apparirà automaticamente nello shop — il codice lo legge dall'array da solo!
⚡ Sfida Extra
Apri la console del browser (F12 → Console) e scrivi: console.log(CATS). Cosa vedi?
La console è come WhatsApp tra te e il codice. Puoi scriverci per vedere cosa contiene ogni variabile.

Missione 4 — Cambia la Password Admin
Impari: JavaScript · Costanti · If/Else
JS ›
🎯 Obiettivo App
La password admin è "admin123" — troppo facile! Cambiala con qualcosa che solo tu sai.
📚 Concetto: Costanti e If/Else
1
const è come let ma non si può cambiare dopo. Perfetto per la password.
2
If/Else = "SE una cosa è vera, FAI questo, ALTRIMENTI fai quest'altro".
3
Il simbolo === significa "è esattamente uguale a". È diverso da = che significa "assegna il valore".
Come funziona il login
JavaScript
// La password è salvata qui:
const PW = 'admin123';  // ← cambia questo

// La funzione di login usa un if/else:
function doLogin() {
  // Prende quello che hai scritto nel form
  const tentativo = document.getElementById('pw').value;

  if (tentativo === PW) {
    // Giusto! Accesso consentito
    isAdmin = true;
  } else {
    // Sbagliato! Mostra errore
    toast('Password sbagliata! ❌');
  }
}
⚡ Sfida Extra
Aggiungi un messaggio personalizzato quando si sbaglia la password. Cambia il testo dentro toast('...') con qualcosa di divertente.

Missione 5 — Capire il Salvataggio Dati
Impari: localStorage · JSON · Dati persistenti
DATA ›
🎯 Obiettivo App
Quando aggiungi un prodotto e ricarichi la pagina, il prodotto è ancora lì. Come funziona? Il browser lo memorizza. Scopriamolo.
📚 Concetto: localStorage e JSON
1
localStorage è come un cassetto nel browser. Puoi mettere dati dentro e restano anche se ricarichi la pagina.
2
JSON è il formato con cui si salvano i dati: è un testo che sembra un oggetto JS. {"nome": "Alessio", "eta": 14}
3
JSON.stringify() = trasforma dati in testo da salvare. JSON.parse() = rilegge il testo e lo ritrasforma in dati.
Prova tu stesso — Console del browser (F12)
Console BrowserF12 → Console
// 1. Salva qualcosa
localStorage.setItem('test', 'ciao mondo');

// 2. Leggilo
localStorage.getItem('test');
// → "ciao mondo"

// 3. Guarda i prodotti di A-HUB!
localStorage.getItem('ahub_v5_products');
// → vedrà tutti i tuoi prodotti in formato JSON

// 4. Trasformali in dati leggibili
JSON.parse( localStorage.getItem('ahub_v5_products') );
// → lista di oggetti, ogni oggetto = un prodotto
🤯 Wow moment
Vedi i tuoi prodotti scritti come oggetti JS? Ogni prodotto ha nome, prezzo, categoria... è così che i siti web memorizzano i dati!
⚡ Sfida Extra
Dalla console scrivi: localStorage.clear() poi ricarica la pagina. Cosa succede? Poi ricarica ancora — da dove vengono i prodotti demo?
Cerca nel codice la variabile DEMO. Questo si chiama "fallback" — un piano B quando non ci sono dati salvati.

Missione 6 — Aggiungi un Bottone
Impari: DOM · Events · onclick · Funzioni
JS ›
🎯 Obiettivo App
Aggiungi un bottone "CONDIVIDI" nella pagina Brand che copia il link dell'app negli appunti. Funzione utile per mandarlo agli amici!
📚 Concetto: DOM ed Events
1
DOM = Document Object Model. È come JavaScript "vede" la pagina HTML: ogni tag è un oggetto che puoi modificare.
2
Un Event è qualcosa che succede: click, scroll, pressione di un tasto. onclick dice "quando clicchi, esegui questa funzione".
3
Una funzione è un blocco di codice con un nome. La chiami e fa la sua cosa.
Codice da aggiungere
HTML + JSsezione Brand nel file
<!-- 1. Aggiungi questo bottone nella sezione brand -->
<button onclick="condividi()" class="prim" style="margin-top:16px">
  📤 Condividi A-HUB
</button>

// 2. Aggiungi questa funzione nello <script>
function condividi() {
  // Copia il link della pagina negli appunti
  navigator.clipboard.writeText(window.location.href);
  toast('Link copiato! Mandalo agli amici 🔗');
}
✅ Risultato
Clicca il bottone → il link di A-HUB viene copiato automaticamente. Hai appena usato una Web API del browser!
⚡ Sfida Extra
Crea una funzione saluta(nome) che mostra toast("Ciao " + nome + "!"). Chiamala dalla console con saluta("Alessio").
Il + tra testo si chiama concatenazione. Unisce pezzi di testo insieme.

Missione 7 — Prodotto del Giorno automatico
Impari: Oggetti JS · Math · Date
JS ›
🎯 Obiettivo App
Mostra un banner "OFFERTA DEL GIORNO 🔥" che evidenzia un prodotto diverso ogni giorno — automaticamente, senza toccare nulla.
📚 Concetto: Oggetti, Math e Date
1
Un Oggetto JS è come una scheda prodotto: ha proprietà con valori. {"nome": "Felpa", "prezzo": 59}
2
Math.floor() arrotonda per difetto. Math.random() genera un numero casuale tra 0 e 1.
3
new Date() crea un oggetto con la data di oggi. .getDay() ti dà il giorno della settimana (0=domenica, 6=sabato).
Codice da aggiungere
JavaScript
// Funzione: prende il "prodotto del giorno"
// Usa il giorno della settimana come indice
function getProdottoDelGiorno() {
  const oggi = new Date().getDay(); // 0-6
  const indice = oggi % products.length; // resta sempre dentro la lista
  return products[indice]; // ritorna il prodotto
}

// Usala così nella pagina shop:
const offerta = getProdottoDelGiorno();
console.log("Offerta oggi: " + offerta.name);

// Il simbolo % è il "modulo" — dà il resto
// della divisione. Es: 8 % 3 = 2
// Serve per non uscire dai limiti dell'array.
Oggetto: dati con etichette
Array: lista ordinata
Indice: posizione in lista (parte da 0!)
% modulo: resto della divisione
⚡ Sfida Extra
Aggiungi un banner giallo in cima alla pagina shop che mostra il nome del prodotto del giorno. Usa document.getElementById() per trovarlo e .textContent = per cambiarne il testo.

Missione 8 — Collegati a Internet (API)
Impari: fetch · API · async/await · JSON da server
PRO ›
🎯 Obiettivo App
Mostra il tasso di cambio €/$ in tempo reale nella sezione Brand. Dati veri presi da Internet, aggiornati ogni secondo.
📚 Concetto: API e fetch
1
Un'API è come un cameriere: tu fai una richiesta (URL), lui porta i dati dal server.
2
fetch(url) è il comando JS per fare una richiesta a Internet e ricevere dati.
3
async/await: siccome Internet ci mette tempo, diciamo al codice di aspettare la risposta prima di continuare.
Esempio funzionante
JavaScriptprova nella console (F12)
// API gratuita di cambio valuta
async function getCambio() {
  // 1. Fai la richiesta all'API
  const risposta = await fetch(
    'https://api.exchangerate-api.com/v4/latest/EUR'
  );

  // 2. Converti la risposta in oggetto JS
  const dati = await risposta.json();

  // 3. Prendi il valore del dollaro
  const dollaro = dati.rates.USD;

  console.log("1€ = $" + dollaro);
  toast("1€ = $" + dollaro.toFixed(2));
}

// Chiamala!
getCambio();
🌐 Questo è il Back-End!
Quello che hai appena fatto è la base del back-end: un server in giro per il mondo ha dei dati, tu li chiedi con fetch(), lui risponde in JSON. Ogni app moderna funziona così — Instagram, TikTok, tutto.
⚡ Sfida Finale
Prova questa API gratuita che dà fatti random sui numeri:
fetch('http://numbersapi.com/42/math')
Poi .text() invece di .json() — perché la risposta è solo testo. Mostralo con toast()!
Alcune API rispondono in JSON (dati strutturati), altre in testo semplice. Devi sapere quale aspettarti.
